Business Problem:¶

Customers, in terms of behavioral shopping patterns, tend to avoid making decisions when exposed to too many buying options. In other words, when a customer is presented with variety of products that exceeds an optimum threshold he/she gets overwhelmed by the number of options. Thus, being able to exploit the recommendation techniques will enable businesses to maximize customer experience. To this end the case study, aims to create a model that is capable of predicting purchase behaviors across differrent business groups.

Method:¶

  • Using a number of tables that contain customer and product attributes, I have carried out an binary classification modelling session. By assembling some of the useful ML pipeline frameworks I have tested some of the well-known ML algorithms which will be presented later.

  • Aside from the genereal information about the customer journey (e.g., product viewing, adding products to basket) on the platform, I have employed Term FrequencyInverse Document Frequency Vectorization (TFIDF) to make full use of the text data by trying to identify the search terms and match them up with the products attributes. This way, I hoped, I could enrich the data at hand to make more accurate predictions.

  • Eventually I aimed to evolve the data into a customer-based format with customer demographics presented along with the user-product interactions.

Zen Score¶

This is a self-crafted feature (with a made up name) that aims to catch a non-linear interaction. In the simplest form, this score indicates if there is a specific bugroupname or bugroupnames that the customer keeps viewing or else is he/she actively wandering around the platform without focusing on a product within a day. This will simply tell the extent of which the user focused on specific kinds of products.

The formula is as follows:

$(1 - views \div total)^2$

$\text{where views = Number of unique products viewed, total = Number of total views made}$

A number close to 1 means a Zen day (e.g., looking for a specific product) and a number closer to 0 means the user does not look for something specific.

Upon calculating the Zen Scores of each user in a given day, now it's time to calculate the number of Zen days and determine if the target currentbugroupname value is in the visit history of the user.

For obvious reasons the accuracy metric is quite deceptive in evaluating the models due to the imbalance between target values. As can be seen from the table, although some of them show better improvements in precision, all of them fail in showing satisfactory results both in f1 score and recall. In addition, the raw probability values of the test output are close to 0.5 which means that the model is not confident with the predictions. The performance of the model is effected by the TFIDF model of course as it left so many search terms unmatched so one solution could be developing a better performing text mining model.

Another and probably more important aspect is to feed non-linear interactions between features. One example can be searching density or in other words measuring the frequency of a user's searching action and calculating a density ratio across all the business group names. However this approach would work better if the date column in search terms dataset would have a higher granularity.

All in all, the model requires a neat work when it comes to feature engineering. This will require great deal of data manipulation and domain knowledge. This pipeline is designed to ease that process and to experiment with novel approaches for coming up with better results.

Challenges:¶

1- Firstly, the data was huge in terms of size, especially for text-mining operations. I wanted use different methods such as TFIDF, word2vec etc. for the product descriptions column. However, the output matrices were made it impossible for my computer to process them. As a solution, I tried to use PySpark framework for distributing the computational load but I failed due to using single node (my local PC) for a huge dataset. I would love to see it work with multiple nodes.

2- I was given 9 different csv files which I thought would be programmatically exhaustive to be merged on Python. Therefore I decided to use SQLite Database on which I could write queries to join multiple tables. For this side I have written a local library that I could manage migrating, accessing, controling an SQL database within a jupyter notebook environment. However SQLite query responses was unexpectedly poor so that I decided to use it for small operations.

3- The given data required some cleaning operations as it had a couple of fields that were provided by humans (i.e., age values provided by customers had values like 2019). I have let PyCaret to take care of the imputation phase but I wonder if KNN Imputer would help increasing performance. Other than that, time granularity of the datasets vary which makes it somewhat hard to come up with an idea to include data details.

Bonus:¶

In the project I have used Ploomber framework to orchestrate every step in an automated fashion. This means that the project can be reproduced by installing MLFlow and Ploomber frameworks described in the Ploomber README file inside the root directory, in case you want to try.

Closing Thoughts¶

I am aware that the models could have performed better with some feature engineering touch. I might have come short in terms of applying domain knowledge to the feature engineering side, due to my lack of experience in e-commerce. However I think that I can come up with a better model as I get more and more experience. Nonetheless I am quite happy with where this project going. Because I wanted this project to be something that can be improved drastically in future. To that end I structured this whole project as an end-to-end pipeline which is the reason why I have integrated Ploomber, PyCaret and MLFlow to the project. So that I can do experiments and figure out the essence of the recommender systems in e-commerce. I have spent a significant amount of time to integrate PySpark in the project which did not end as hoped but I learned a lot even when failing. Overall I have tried to accomplished so much in so little time and I enjoyed the journey.
